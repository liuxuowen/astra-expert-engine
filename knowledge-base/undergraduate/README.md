# 本科申请知识库

按国家/地区分类的本科申请院校速查、顾问话术与导入规范沉淀。

## 目录结构

```text
undergraduate/
├── README.md
├── usa/          # 美国本科
├── uk/           # 英国本科
├── sg/           # 新加坡本科
├── hk/           # 中国香港本科
├── au/           # 澳大利亚本科
├── ca/           # 加拿大本科
├── eu/           # 欧洲（德国、荷兰、法国等）
└── other/        # 其他国家/地区
```

## 文件组织原则
- 每个国家/地区对应一个子目录
- 每所院校原则上一个主文件，沉淀该校的本科申请核心信息
- 院校文件建议命名为：`{school}-undergraduate-{season}.md`
  - 例：`nus-undergraduate-2026-2027.md`
  - 例：`hkust-undergraduate-2026-2027.md`
- 同一所学校的新增信息，优先追加到既有主文件，不轻易拆成多个碎片文件

## 单校文件推荐结构
每个院校主文件建议按以下层次组织：

1. **顾问可直接对外讲的咨询话术版**
2. **三段速查版**
   - 申请要求
   - 奖学金
   - 国际生适配
3. **家长沟通版话术**
4. **学生沟通版话术**
5. **知识库使用建议 / 后续扩展建议**

## 导入体系（按本次对话逻辑固化）
后续投喂院校 PDF、截图、网页或文字内容时，统一按以下流程导入：

### Step 1：识别基础归类
先判断以下三件事：
- **国家/地区**：写入对应子目录（如 `sg/`、`hk/`、`uk/`）
- **院校**：并入该院校主文件
- **阶段**：本科内容统一进入 `knowledge-base/undergraduate/`

### Step 2：提炼四层内容
导入时，优先从原始材料中提炼以下四层：
1. **事实层**：学校排名、项目设置、官方要求、申请流程、奖学金规则
2. **判断层**：哪些信息值得顾问重点关注，哪些适合咨询口径复用
3. **话术层**：整理成能直接对家长/学生说的话
4. **边界层**：哪些不能说满，哪些要提醒“以官网为准”

### Step 3：写入单校主文件
原则上按以下顺序写入：
- 先更新“顾问可直接对外讲的咨询话术版”
- 再更新“三段速查版”
- 再补“家长沟通版 / 学生沟通版”
- 最后补充使用边界、后续待扩展项

### Step 4：避免碎片化
- 不为同一所学校的单一 PDF 单独建一个文件
- 同校多份官方材料，优先合并整理进一个主文件
- 只有当内容明显独立且持续增长时，才考虑拆子文件

## 命名与写作规范
- 用中文写作，保留必要英文项目名称
- 先给结论，再给解释
- 尽量形成可直接复用的话术，而不是单纯摘抄材料
- 奖学金、补贴、financial aid、bond / service obligation 必须区分清楚
- 涉及时间、金额、政策、申请节点时，尽量注明“对应申请季”
- 若内容来自官方 brochure / prospectus，默认可作为高可信信息，但仍建议保留“以官网最新要求为准”的边界表达
